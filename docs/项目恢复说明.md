# CodeQuest 项目恢复说明

## 1. 文档目的
本文档用于说明如何在新环境中恢复并运行 CodeQuest 项目，覆盖：
- 使用的数据库类型与数据库名
- 开发环境与运行环境
- 初始账号信息
- 全量恢复与题库单独恢复步骤
- 启动后验证方法

---

## 2. 项目与环境信息

### 2.1 项目名称
CodeQuest（Servlet/JSP 面试练习系统）

### 2.2 开发环境
- 操作系统：Windows 10/11
- JDK：17
- Maven：3.8+
- IDE：IntelliJ IDEA / Eclipse（任一）
- 数据库：MySQL 8.0
- 缓存：Redis 6+
- 容器：Docker Desktop（Compose v2）

### 2.3 运行环境
- 推荐运行方式：Docker Compose（MySQL + Redis + Tomcat 一键启动）
- 应用访问地址：http://localhost:8081
- MySQL 端口：3306
- Redis 端口：6379

### 2.4 数据库信息
- 数据库类型：MySQL
- 数据库名：codequest_db
- 字符集：utf8mb4

---

## 3. 恢复前准备

### 3.1 必备软件
1. 安装 JDK 17 并配置 JAVA_HOME
2. 安装 Maven（若未安装，可通过 IDE 自带 Maven）
3. 安装 Docker Desktop（确保 docker compose 可用）

### 3.2 启动前检查
在仓库根目录执行以下命令（PowerShell）：

```powershell
docker --version
docker compose version
java -version
```

---

## 4. 项目恢复与启动步骤

### 步骤 1：编译项目
进入 platform 目录后执行：

```powershell
cd .\platform
mvn clean package -DskipTests
```

### 步骤 2：启动容器
本项目 Compose 入口在仓库根目录。可二选一：

方式 A（从 platform 返回根目录）：
```powershell
cd ..
docker compose up -d
```

方式 B（在根目录直接执行）：
```powershell
docker compose up -d
```

### 步骤 3：查看服务状态
```powershell
docker compose ps
```

预期：
- codequest-mysql 为 healthy
- codequest-redis 为 healthy
- codequest-app 为 up

## 5. 数据库恢复方式

## 5.1 全量恢复（推荐用于新环境）
用途：恢复核心表结构 + 初始用户 + 初始题库 + 提示词模板。

脚本：platform/scripts/recover_codequest_db.sql

执行方式（PowerShell）：
```powershell
docker cp .\platform\scripts\recover_codequest_db.sql codequest-mysql:/tmp/recover_codequest_db.sql
docker exec codequest-mysql sh -lc "mysql -uroot -proot < /tmp/recover_codequest_db.sql"
```

也可一键执行：
```powershell
.\platform\scripts\run-recover-db.ps1 -SqlFile .\platform\scripts\recover_codequest_db.sql
```

## 5.2 题库单独恢复（仅恢复题目数据）
用途：仅覆盖题库相关两张表（t_question、sys_question）。

脚本：platform/scripts/recover_question_bank.sql

执行方式（PowerShell）：
```powershell
docker cp .\platform\scripts\recover_question_bank.sql codequest-mysql:/tmp/recover_question_bank.sql
docker exec codequest-mysql sh -lc "mysql -uroot -proot codequest_db < /tmp/recover_question_bank.sql"
```

注意：
- 该脚本包含 DROP TABLE IF EXISTS，会覆盖当前两张题库表数据。

## 8. 常见问题与处理

### 问题 1：启动时容器名冲突
现象：提示 container name already in use。

处理：
```powershell
docker rm -f codequest-app codequest-mysql codequest-redis
docker compose up -d
```

### 问题 2：数据库中文乱码
处理建议：
- 确认数据库和连接字符集使用 utf8mb4
- SQL 文件使用 UTF-8 编码

### 问题 3：mvn 命令不可用
处理建议：
- 安装 Maven 并配置环境变量
- 或在 IDE 内使用 Maven 面板执行 clean/package

### 问题 4：DeepSeek Key 配置
建议通过环境变量传入，不要明文写入仓库。

PowerShell 示例：
```powershell
$env:DEEPSEEK_API_KEY="your_deepseek_api_key"
```
